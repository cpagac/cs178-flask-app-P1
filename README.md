## Overview

The Turing Test is a web-based game where players attempt to distinguish between text written by humans and text generated by AI models. This project implements an "Arena" where all snippets from the database are displayed, allowing users to see the source of various content across categories like Python code, poetry, and news articles.

---

## Technologies Used

- **Flask** — Python web framework
- **AWS EC2** — hosts the running Flask application
- **AWS RDS (MySQL)** — relational database for storing code/text snippets, categories, and author metadata.
- **AWS DynamoDB** — (Implementation pending/for user management)
- **GitHub Actions** — auto-deploys code from GitHub to EC2 on push

## How to Run Locally

1. Clone the repository:

   ```bash
   git clone https://github.com/cpagac/cs178-flask-app-P1
   cd cs178-flask-app-P1
   ```

2. Install dependencies:

   ```bash
   pip3 install flask pymysql cryptography
   ```

3. Set up your credentials:
   Create a `creds.py` file in the root directory with your RDS endpoint and credentials.

4. Seed the database:
   ```bash
   mysql -h your-rds-endpoint -u admin -p < schema.sql
   ```

5. Run the app:

   ```bash
   python3 flaskapp.py
   ```

6. Open your browser and go to `http://127.0.0.1:8080/arena`

## Database Design

### SQL (MySQL on RDS)

- `Categories` — stores types of content (e.g., Python Code, Poetry); primary key is `id`
- `Authors` — stores the name of the author and whether they are AI; primary key is `id`
- `Snippets` — stores the actual text content; foreign keys link to `category_id` and `author_id`

The **JOIN query** used in this project retrieves all snippets and connects them with their specific Category name and Author details (including the `is_ai` flag) to display a complete record in the Arena.
